🏌️ 스크린골프 관리 시스템

웹서비스 배포 & 구글 로그인 연동 실행 가이드

📌 이 문서는 로컬 개발 환경에서 완성된 앱을 인터넷(Vercel)에 배포하고, 구글 계정 로그인 기능을 연동하는 전체 과정을 단계별로 정리한 요약본입니다. 처음부터 차례대로 따라하면 누구나 동일한 결과를 재현할 수 있습니다.

TASK 1. 웹서비스 배포 (GitHub + Vercel)

사전 준비

  • Git 설치 완료 : https://git-scm.com
  • GitHub 계정 보유 : https://github.com
  • Vercel 계정 보유 : https://vercel.com (구글 계정으로 가입 가능)

Step 1 — 프로젝트를 Git 저장소로 초기화

VS Code 터미널을 열고 프로젝트 폴더에서 아래 명령어를 순서대로 입력합니다.

git init
git add .
git commit -m "first commit"
💡 이 작업은 최초 1회만 수행합니다. 프로젝트 폴더에 Git 관리 기능을 장착하는 단계입니다.

Step 2 — GitHub에 원격 저장소(Repository) 생성

  1. GitHub에 로그인합니다.
  2. 우측 상단 [+] 버튼 → [New repository]를 클릭합니다.
  3. Repository name에 원하는 이름을 입력합니다. (예: Prize-Management)
  4. [Create repository] 버튼을 눌러 저장소를 생성합니다.
  5. 생성된 저장소의 주소(URL)를 복사합니다.
    • 예시: https://github.com/계정명/Prize-Management.git

Step 3 — 로컬 프로젝트와 GitHub 저장소 연결

VS Code 터미널에서 아래 명령어를 입력합니다. (2번에서 복사한 본인의 GitHub 주소로 교체하여 입력)

git remote add origin https://github.com/계정명/Prize-Management.git
git branch -M main
git push -u origin main
💡 처음 push 시 GitHub 로그인 창이 팝업될 수 있습니다. 계정 정보를 입력하여 인증을 완료합니다.

Step 4 — Vercel에 프로젝트 연결 및 환경변수 설정

  1. Vercel에 로그인합니다.
  2. 대시보드에서 [Add New Project]를 클릭합니다.
  3. GitHub 계정과 연동 후, 방금 생성한 저장소(Prize-Management)를 선택합니다.
  4. [Environment Variables (환경변수)] 섹션에서 아래 두 가지 키를 입력합니다.
변수명 값
VITE_SUPABASE_URL Supabase 프로젝트 URL
VITE_SUPABASE_ANON_KEY Supabase 익명(anon) API 키
💡 위 두 값은 Supabase 대시보드 → [Project Settings] → [API] 탭에서 확인합니다.
  1. [Deploy] 버튼을 클릭합니다.
  2. 약 1~2분 후 배포가 완료되면, Vercel이 고유한 웹 주소를 자동으로 발급해 줍니다.
    • 예시: https://02-prize-manager.vercel.app

✅ 이제 이 주소로 누구나 인터넷에서 접속 가능합니다!

✅ 앞으로 git push를 할 때마다 Vercel이 자동으로 최신 버전으로 갱신됩니다.

TASK 2. 구글 로그인 연동 (Google OAuth + Supabase)

사전 준비

  • Google 계정 보유
  • Google Cloud Console 접속 가능 : https://console.cloud.google.com
  • Supabase 프로젝트 대시보드 접속 가능

Step 1 — Google Cloud Console에서 OAuth 키 발급

[1-1] 접속 및 프로젝트 확인

  1. 구글 클라우드 콘솔에 접속하여 구글 계정으로 로그인합니다.
  2. 상단 프로젝트 선택 버튼을 눌러 사용할 프로젝트를 선택하거나 새로 만듭니다.

[1-2] OAuth 동의 화면 구성

  1. 상단 검색창에 "OAuth 동의 화면"을 검색하여 해당 메뉴로 이동합니다.
  2. [시작하기] 버튼을 클릭합니다.
  3. 아래 항목을 입력합니다.
    • 앱 이름: 스크린골프 관리 (원하는 이름)
    • 사용자 지원 이메일: 본인 구글 이메일 선택
    • 개발자 연락처: 본인 구글 이메일 입력
  4. [저장하고 계속] 버튼을 반복해서 눌러 모든 단계를 완료합니다.

[1-3] OAuth 클라이언트 ID 생성

  1. 왼쪽 메뉴에서 [클라이언트]를 클릭합니다.
  2. [+ 클라이언트 만들기] 버튼을 클릭합니다.
  3. 애플리케이션 유형: [웹 애플리케이션]을 선택합니다.
  4. [승인된 리디렉션 URI] 섹션에서 [+ URI 추가]를 누르고 아래 주소를 입력합니다. 
    https://[본인의 Supabase 프로젝트 ID].supabase.co/auth/v1/callback
    💡 Supabase 프로젝트 ID는 Supabase 대시보드 URL에서 확인합니다.
    ※ 예시: https://zpqofdfvqrpjuwcflhqi.supabase.co/auth/v1/callback
  5. [만들기] 버튼을 클릭합니다.
  6. 팝업 창에 나타나는 [클라이언트 ID]와 [클라이언트 보안 비밀(Secret)]을 안전한 곳에 복사해 둡니다.

Step 2 — Supabase에 구글 Provider 등록

  1. Supabase 대시보드에 로그인합니다.
  2. 해당 프로젝트 → 왼쪽 메뉴 [Authentication] → [Providers]로 이동합니다.
  3. 목록에서 [Google]을 찾아 클릭합니다.
  4. [Enable Sign in with Google] 스위치를 ON으로 켭니다.
  5. 아래 두 칸에 Step 1에서 복사해 둔 값을 붙여넣습니다.
입력 칸 붙여넣을 값
Client IDs 구글에서 복사한 클라이언트 ID
Client Secret (for OAuth) 구글에서 복사한 클라이언트 보안 비밀
  1. [Save] 버튼을 클릭합니다.
  2. Google 항목에 [Enabled] 초록색 뱃지가 표시되면 성공입니다.

Step 3 — Supabase URL 설정 (로컬 + 실서버 동시 허용)

⚠️ 주의: 이 단계를 빠뜨리면 실서버(Vercel)에서 로그인 후 localhost로 튕기는 오류가 발생합니다.

  1. Supabase 대시보드 → [Authentication] → [URL Configuration]으로 이동합니다.
  2. [Site URL] 칸의 값을 실서버 주소로 변경합니다.
    • 입력값: https://02-prize-manager.vercel.app
  3. [Redirect URLs] 섹션에서 [Add URL]을 눌러 로컬 주소를 추가합니다.
    • 입력값: http://localhost:5173/
  4. [Save] 버튼을 클릭합니다.

✅ 이제 실서버와 로컬 개발 환경, 두 곳 모두에서 구글 로그인이 정상 작동합니다!
